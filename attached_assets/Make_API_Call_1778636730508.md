# Make_API_Call.md
## How the Tableicity Replit pulls published HTML from Halo

**Audience:** the operator/agent on the Tableicity Replit (and later, the Investor Ensights Replit, and any future sister site).
**Direction:** Tableicity pulls *from* Halo. Halo never pushes. Halo never receives anything from Tableicity.
**What you get:** finished, minified HTML for a single persona's published items, plus an `id` you use to track what you've already pulled.

---

## The handshake in one sentence

> Hold an API key in a Replit Secret. Every five minutes, GET one URL with that key as a Bearer token, passing the highest `id` you've seen so far. Import whatever comes back. Save the new highest `id`. That's the entire integration.

---

## What John gives you

You will receive **one string** from John that looks like this:

```
halo_AbCdEf1234567890XYZqRsTuVwXyZ123abc=
```

That is the API key for **Tableicity only**. It is bound to the Tableicity persona on the Halo side. You cannot use it to fetch Vectra content, Reel content, or any other persona's content. There is no parameter for "which persona" — the key *is* the persona selector. This is intentional.

**John can only show this key to you once.** If you lose it, John must rotate it on the Halo Distribution page, which produces a new key and instantly kills the old one.

---

## Step 1 — Store the key as a Replit Secret

In the Tableicity Replit:

1. Open the Secrets pane (lock icon in the sidebar).
2. Add a new secret:
   - **Key:** `HALO_DISTRIBUTION_KEY`
   - **Value:** the `halo_…` string from John
3. Save.

Do **not** paste the key into any source file. Do **not** commit it. Treat it like a password.

---

## Step 2 — Pick a place to remember the high-water mark

You need to remember the largest `id` you've already pulled, so you don't re-import the same article twice. Use whatever storage Tableicity already has:

- A row in the Tableicity database (e.g. `halo_sync.last_id`), or
- A single-cell value in object storage, or
- A WordPress option / meta value if you're writing direct to WP.

The first time you ever run, treat the stored value as `0`.

---

## Step 3 — The single endpoint

```
GET https://haylords.com/api/public/published?since=<lastId>
Authorization: Bearer <HALO_DISTRIBUTION_KEY>
```

That's it. There is no other endpoint. There is no `POST`, no `DELETE`, no auth-token-exchange step.

**Response:** a JSON array, sorted ascending by `id`, capped at 100 items per call.

```json
[
  {
    "id": 4123,
    "publishedAt": "2026-05-13T14:22:01.000Z",
    "htmlContent": "<h1>Your Article Title</h1><p>…minified body…</p>"
  },
  {
    "id": 4124,
    "publishedAt": "2026-05-13T14:48:11.000Z",
    "htmlContent": "<h1>Next Article</h1><p>…</p>"
  }
]
```

Each `htmlContent` is **already** the finished, scrubbed, persona-voiced HTML. The `<h1>` at the top is the title (Halo prepends it to the body during render). If a piece had Answer Blocks generated, you'll see `<p class="answer-block" itemprop="abstract">…</p>` paragraphs inside — leave them alone, they help SearchGPT/Perplexity/Gemini cite the article.

## Step 4 — What "done" looks like for one cycle

Pseudo-code (use whatever language Tableicity is in):

```ts
const HALO_KEY = process.env.HALO_DISTRIBUTION_KEY!;
const HALO_URL = "https://haylords.com/api/public/published";

let lastId = await getStoredHighWaterMark();   // 0 the first time

while (true) {
  const res = await fetch(`${HALO_URL}?since=${lastId}`, {
    headers: { Authorization: `Bearer ${HALO_KEY}` },
  });

  if (res.status === 401) throw new Error("Halo rejected the key — ask John to rotate.");
  if (res.status === 429) { await sleep(60_000); continue; } // back off one minute
  if (!res.ok) throw new Error(`Halo returned ${res.status}`);

  const items = await res.json();
  if (items.length === 0) break;               // caught up

  for (const item of items) {
    await importToTableicity(item.id, item.publishedAt, item.htmlContent);
  }

  lastId = items[items.length - 1].id;
  await storeHighWaterMark(lastId);

  if (items.length < 100) break;               // last page, no need to loop again
}
```

Run that whole block on a schedule — every 5 minutes is plenty. There is no harm in running it more often, but you'll just hit empty responses.

---

## Step 5 — On the Tableicity import side

Whatever `importToTableicity(id, publishedAt, htmlContent)` does is **your** call. Common patterns:

- Push directly into a WordPress post via the WP REST API, with `htmlContent` as the post body and `publishedAt` as the post date.
- Stage it into a Tableicity DB table for human review before publish.
- Drop it into an `imported/` bucket in object storage and let your existing pipeline pick it up.

**Important:** key on `id`. If you ever see the same `id` twice, treat the second one as a no-op — it shouldn't happen if your high-water mark is honest, but defensive code is cheap.

## Rules Halo enforces (so you don't get surprised)

| Rule | What it means for you |
|---|---|
| **Read-only** | The endpoint only does `GET`. No write/delete/update path exists. |
| **Persona-locked by key** | Your key returns Tableicity items only. Period. |
| **Page size 100** | Loop until you get fewer than 100 items back. |
| **Rate limit: 60 requests/minute per key** | Way more than you need. If you hit 429, sleep a minute and retry. |
| **Includes `published` and `archived`** | Halo keeps archived items in the feed for audit/E-E-A-T. The `id` keeps you idempotent, so this doesn't cause double-imports. |
| **Sorted ASC by id** | `since=<lastId>` returns items strictly greater than `lastId`. The last item in the array has the highest id. |
| **Key never in URL** | Always Bearer header. Never query string. URL logs are not safe places for secrets. |

---

## When something goes wrong

| Symptom | Meaning | What to do |
|---|---|---|
| `401 Missing or malformed Authorization header` | You forgot the `Bearer ` prefix or the header. | Fix the header. |
| `401 Invalid or revoked API key` | Key is wrong, was rotated, or was revoked on the Halo side. | Ask John to confirm/rotate, then update the Replit Secret. |
| `429 Rate limit exceeded` | You're polling too hard. | Sleep 60 seconds and resume. |
| `429 Too many requests` | Pre-auth IP throttle (32 attempts in a minute from one IP, usually means many bad keys). | Stop and check that the key is loaded correctly. |
| Empty array `[]` | You're caught up. Nothing new since `lastId`. | Sleep until next scheduled run. |
| `5xx` | Halo is briefly down. | Retry with exponential backoff (e.g. 30s, 1m, 2m, 5m). Do **not** advance the high-water mark. |

---

## What John sees on his side

When you call the endpoint successfully, Halo updates two columns on your consumer record:
- **Last Pulled** — timestamp of your most recent successful call.
- **Delivered** — running count of items handed to you.

John watches those two numbers on the Halo **Distribution** page. If "Last Pulled" goes stale, he'll know your cron stopped. If "Delivered" jumps as expected, the handshake is healthy.

## Quick sanity check (do this first, before wiring the cron)

Open a shell on the Tableicity Replit and run:

```bash
curl -H "Authorization: Bearer $HALO_DISTRIBUTION_KEY" \
  "https://haylords.com/api/public/published?since=0" | head -c 500
```

You should see a JSON array starting with `[{"id":…,"publishedAt":…,"htmlContent":"<h1>…`.

If you do — the handshake works. Wire the cron. You're done.

If you don't — stop and report exactly which symptom from the table above you hit. Don't guess; ask John.
